Object Creation Patterns in JavaScript

how to create confusion in the mind of a software developer

The factory creation pattern

Or how to abstract the creation of complex objects...


function pixelFactory(x, y, color) {
  return {
    x: x,
    y: y,
    color: color
  };
}

function circleFactory(point, radius) {
  return {
    point: point,
    radius: radius,
    shrink : function(value) {
      this.radius = this.radius - value > 0 ? this.radius - value : 0;
    }
  };
}

var p1 = pixelFactory(2,3,'red');
var p2 = pixelFactory(6,2,'green');
var c1 = circleFactory(p1, 10);
var c2 = circleFactory(p2, 10);
c1.shrink(2);
c2.shrink(1);
One major drawback... memory usage is very inneficient
Compare to classical OOP

The constructor pattern

Or how to emulate classical inheritance in javascript



function Pixel(x, y, color) {
  this.x = x;
  this.y = y;
  this.color = color;
}

function Circle(pixel, radius) {
  this.pixel = pixel;
  this.radius = radius;
  this.shrink = function(value) {
    this.radius = this.radius - value > 0 ? this.radius - value : 0;
  };
}

var p1 = new Pixel(1,2,'red');
var p2 = new Pixel(6,2,'green');
var c1 = new Circle(p1,10);
var c2 = new Circle(p2,10);
c1.shrink(2);
c2.shrink(1);

DO NOT FORGET THE NEW OPERATOR!!!!

by convention, constructor functions are capitalized
new creates a new object using the provided constructor function
new object is accessible to the constructor function with this

Prototypal Inheritance in JavaScript

Prototype-based programming is an OOP model that doesn't use classes, but rather accomplishes behavior reuse by expanding upon existing prototype objects.


var sleeper = {
  sleep: function() {
    console.log('Sleeping...');
  }
};

function Programmer(type) {
  this.type = type;
  this.speak = function(line) {
    console.log('The ' + this.type + ' programmer says ' + line);
  };
}

var javaScriptProgrammer = new Programmer('JavaScript');
javaScriptProgrammer.speak('I know prototypes!');
javaScriptProgrammer.sleep();

Programmer.prototype = sleeper;

var javaProgrammer = new Programmer('Java');
javaProgrammer.speak('I know reflection!');
javaProgrammer.sleep();

Constructor/Prototype Pattern

Effecient memory usage due to one time creation of shrink function in Circle prototype


function Circle(pixel, radius) {
  this.pixel = pixel;
  this.radius = radius;

Circle.prototype.shrink = function(value) {
  this.radius = this.radius - value > 0 ? this.radius - value : 0;
